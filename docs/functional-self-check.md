# HM-CLSS 功能级自测清单

这份清单用于在结构调整、交互改版或新流程上线后做一轮更严格的人工验收。建议顺序是：

1. 先跑静态巡检：`bash scripts/smoke-check.sh`
2. 再跑浏览器冒烟：`bash scripts/browser-smoke.sh`
3. 最后按下面的功能级清单人工过一遍

## 启动与外壳

- 页面首屏能正常渲染，顶部时间不是空白。
- 深浅色切换后，图标、卡片、图表颜色和下拉框风格都同步变化。
- 左侧导航切舱时，主面板会回到顶部，不会保留上一个页面的滚动位置。
- 首屏主说明区会随当前舱段同步变化，不出现错位文案。

## 导航与快捷键

- `Alt+1-8` 能切到对应舱段。
- `Alt+[` 和 `Alt+]` 能按顺序循环切换舱段。
- `Ctrl+K` 能在任意非输入态下打开捕捉台，再按一次或按 `Esc` 能关闭。
- 输入框、文本域或下拉框聚焦时，导航快捷键不会误触。

## 值班与补打卡

- 早 / 中 / 晚三个班次的实时按钮会随当前时段和离舰状态正确启停。
- 实时打卡后，今日状态卡、值班表和统计口径会同步刷新。
- 提前上班卡显示为 `警告`，不是 `不合格`。
- 补打卡只能选最近 30 天，超窗时按钮保持禁用并给出明确提示。
- 补打卡在 7 天 / 自然月配额触顶时不能提交。
- 补打卡提交后，配额、留痕和相关统计会同步变化。
- 全天离舰存在时，补打卡面板会阻止提交并提示先回离舰流程调整。

## 离舰流程

- `今日离舰 / 预请假 / 补请假` 三个入口的标题、说明、字段锁定状态完全对应。
- 今日离舰只收今天；预请假只收未来日期；补请假只收过去日期。
- 补请假缺少补录说明时不能提交。
- 历史日期已有旧请假或旧打卡时，补请假会提示二次确认风险。
- 撤销离舰记录后，今日按钮和首页状态会立刻恢复。

## 任务与捕捉池

- 任务开始、结束、恢复后，当前任务区、今日日志和时间轴都同步更新。
- 深度工作和捕捉池的卡面高度、按钮对齐保持稳定，不出现明显空洞。
- 捕捉台保存后，今天卡片和归档检索都能立刻看到新记录。
- 归档搜索为空、命中、多条命中三种状态都能正常显示。

## 调酒吧台

- 输入为空时，分析按钮保持禁用。
- 情绪文本输入后，分析态能正常切到结果态，不会卡死在中间。
- 酒液波面在深浅色模式下都正常显示，不出现漏色、椭圆块或纯色遮挡。
- 结果卡左侧信息不裁切，历史酒单可进入、返回、再创建。
- 保存到酒柜后，历史数量会递增，彩蛋酒款不会破坏布局。

## 统计与同步

- 统计周期切换后，卡片摘要和图表一起刷新。
- 暗黑模式切换后，图表不会丢色或残留浅色文本。
- GitHub Token / Gist ID 输入区和上传 / 拉取按钮在设置页都能正常聚焦和点击。
- 在未配置同步凭据时，相关操作会给出明确提示，不会静默失败。

## 回归收口

- 桌面端首屏能直接看到主动作，不需要先滚到卡片底部。
- 常用卡片内部不应再出现必须额外滚动才能触达主操作的情况。
- 所有系统下拉框在浅色和暗黑模式下都保持统一视觉。
- 页面不存在大块纯色遮挡、卡片越界或滚动穿帮。
